[NOI 2014] 动物园

Posted on 2018-01-25 Comments:

题目大意

给定一个长为 n 的字符串 S ，定义 num[i] 为在串 S[1:i] 中，既为它的前缀，又为它的后缀，且前缀与后缀不重叠的子串个数。求出 $ \prod num[i] ,, mod ,, 1000000007$ 。

解题报告

观察num[i] 定义：对于 S[1:i] ，既为它的前缀，又为它的后缀，且前缀与后缀不重叠的子串个数。
因为“前缀与后缀不重叠”这个条件不容易判断，那么就先忽略。

令 f[i] 表示对于 S[1:i] ，既为它的前缀，又为它的后缀的子串个数（包括前缀和后缀都为自身的情况），则有
　f[1] = 0
　f[i] = f[ P[i] ] + 1，其中 P[i] 即为 KMP 中的 next[i]
　原因：
　　求 f 实质上就是遍历 S[1:i] 中的所有子串 S[1:j] ，若 S[1:j] = S[i-j+1:i] ，则算入 f 中。我们来倒序遍历子串：
　　首先，最大的子串，即自身 S[1:i] = S[1:i] 的情况要算进去；
　　然后要计算的第二大的子串，也就是 P[i] 所表示的 S[1:P[i]] = S[i-P[i]+1:i]。而之后的所有其他子串都是在 S[1:P[i]] 之内的，那么就可以用 f[ P[i] ] 递推得出。

再令 g[i] = num[i] ，g[i] 可以由 f 算出。
g[i] 类似于 f[i] ，只不过在子串 S[1:j] 上加了一个限制 j <= i/2 。
那么在求解时，先令 j = P[i] ，然后一直令 j = P[j] ，直到 j 满足限制条件，那么最终 f[j] 即为 g[i] 的值。

但如果这样计算，当 S=”aaaaa…” 时，[1] 处的代码就会变成 $O(n)$ 的，整体的复杂度就被卡到了 $O(n^2)$ ，因此要想办法优化。

可以看出，对于 g[i] 的求解实际上就是找出最大的 j ，使得 S[1:j] = S[i-j+1:i] 并且 j <= i/2 。而这样的过程与 KMP 中 next 数组的计算十分接近，只不过多了一个 j <= i/2 的条件。那么就类似 next 数组的求解，将 S 与自身配对求解 g 。

而这样做的复杂度是线性的。
因为只有在 [2] 处 j 可能增加，在 [1]、[3] 处 j 都是减小的。又因为 j 最多只增加了 n ，那么最多只能减小 n 次，于是复杂度就是线性的了。

总结：

  1. 一开始没有把限制条件去掉，结果怎么弄都不行。
  2. 在考虑限制条件时，一开始想的是顺序遍历 P 然后使得每个 P[i] 都满足 P[i] <= i/2，然而这样会影响到之后的 P 值，只能倒序求解，但这样就会被卡到 $O(n^2)$ 。
  3. 现在感觉 KMP 最精华的就是 next 数组，还有计算它时“匹配自己”的方法。
  4. 代码竟然这么短……
  5. 仿佛 NOI 的题目不是很难嘛 XD
